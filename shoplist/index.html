5. Функция совместного доступа к спискам

Приложение позволяет делиться списком покупок с другим пользователем по его адресу электронной почты. При использовании этой функции:

  • Указанный вами email-адрес используется для поиска аккаунта получателя в Firestore.
  • Получателю предоставляется доступ к содержимому списка.
  • Вы несёте ответственность за то, что передаёте данные только тем лицам, которые дали на это согласие.

6. Хранение данных

Локальное хранилище

Списки и позиции хранятся в базе данных SQLite на вашем устройстве. Данные удаляются при удалении Приложения.

Облачное хранилище (Firebase)

Данные хранятся на серверах Google Firebase до тех пор, пока существует ваш аккаунт.

Удаление аккаунта

При удалении аккаунта через Приложение ваш аккаунт Firebase Auth деактивируется. Для полного удаления облачных данных из Firestore направьте запрос на адрес sasha.robson@gmail.com — данные будут удалены в течение 30 дней.

9. Безопасность данных

  • Аутентификация защищена сервисом Firebase Auth: пароли хранятся только в зашифрованном виде.
  • Передача данных между устройством и Firebase осуществляется по протоколу HTTPS/TLS.
  • Доступ к данным в Firestore ограничен правилами безопасности: пользователь может читать и изменять только свои собственные данные и те списки, к которым ему был предоставлен доступ явным образом.
  • FCM-токен хранится только в вашем профиле Firestore и не передаётся третьим лицам.

Ни один способ передачи и хранения данных в интернете не является абсолютно защищённым. Мы не можем гарантировать 100 % безопасность.
